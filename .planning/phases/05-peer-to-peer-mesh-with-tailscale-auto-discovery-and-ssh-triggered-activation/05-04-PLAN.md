  <how-to-verify>
**Prerequisites:**
1. Install tassh on both local and remote machines:
   ```bash
   # On both machines:
   cargo install --path .
   ```

2. Run setup on both machines:
   ```bash
   # On both machines:
   tassh setup daemon
   ```

3. Verify daemons are running:
   ```bash
   # On both machines:
   systemctl --user status tassh-daemon
   ```

**Test 1: Automatic connection on SSH**

1. On local machine, take a screenshot (Ctrl-Shift-Print or your screenshot tool)

2. SSH to the remote machine:
   ```bash
   ssh remote-hostname
   ```

3. Check that connection was established:
   ```bash
   # On local machine (new terminal):
   tassh status
   ```
   Expected: Shows `remote-hostname - connected (1 SSH session)`

4. On remote, verify clipboard has the screenshot:
   ```bash
   # On remote (in SSH session):
   xclip -selection clipboard -t image/png -o > /tmp/test.png
   file /tmp/test.png
   ```
   Expected: `PNG image data`

**Test 2: Claude Code paste**

1. On local, take a new screenshot

2. In SSH session, start Claude Code:
   ```bash
   claude
   ```

3. Press Ctrl-V in Claude Code
   Expected: Shows `[Image #1]` or similar image indicator

**Test 3: Multiple sessions share connection**

1. Open a second SSH session to the same host:
   ```bash
   ssh remote-hostname
   ```

2. Check status:
   ```bash
   tassh status
   ```
   Expected: Shows `remote-hostname - connected (2 SSH sessions)`

3. Close one SSH session (Ctrl-D or exit)

4. Check status again:
   ```bash
   tassh status
   ```
   Expected: Still connected with 1 session

5. Close the last SSH session

6. Check status (may take a moment for pidfd to detect):
   ```bash
   tassh status
   ```
   Expected: No active connections (or connection gone)

**Test 4: No daemon on remote (graceful)**

1. Stop daemon on remote:
   ```bash
   # On remote:
   systemctl --user stop tassh-daemon
   ```

2. SSH to remote:
   ```bash
   ssh remote-hostname
   ```
   Expected: SSH connects normally (no delay or error)

3. Check status:
   ```bash
   tassh status
   ```
   Expected: May show `remote-hostname - probing` briefly, then no connection (no daemon found)

4. Restart daemon on remote:
   ```bash
   # On remote:
   systemctl --user start tassh-daemon
   ```
  </how-to-verify>

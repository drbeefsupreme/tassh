<task type="auto">
  <name>Task 2: Implement transport module and wire into main</name>
  <files>src/transport.rs, src/main.rs</files>
  <action>
1. Implement src/transport.rs with these components:

   **TransportError enum** (using thiserror):
   - `Io(#[from] std::io::Error)` — wraps IO errors
   - `WriteTimeout` — 10-second write deadline exceeded
   - `ConnectionClosed` — peer closed cleanly (read_exact UnexpectedEof)
   - `Frame(#[from] crate::protocol::FrameError)` — frame parse errors

   **apply_keepalive(stream: &tokio::net::TcpStream) -> io::Result<()>**:
   - Use socket2::SockRef::from(stream) and TcpKeepalive::new()
   - Configure: time=10s, interval=5s, retries=3 (detection within ~25s, under 30s requirement)

   **send_frame(writer: &mut OwnedWriteHalf, frame: &Frame) -> Result<(), TransportError>**:
   - Serialize frame via frame.to_bytes()
   - Wrap write_all in tokio::time::timeout(Duration::from_secs(10), ...)
   - Map Elapsed to TransportError::WriteTimeout
   - Map io::Error to TransportError::Io

   **recv_frame(reader: &mut OwnedReadHalf) -> Result<Frame, TransportError>**:
   - read_exact 8-byte header (HEADER_LEN from protocol.rs — use the constant, it's pub(crate) or define locally as 8)
   - On UnexpectedEof: return TransportError::ConnectionClosed
   - Parse payload length from header bytes [4..8] as u32 big-endian
   - read_exact payload bytes
   - Combine header + payload, parse via Frame::from_bytes

   **pub async fn server(bind_addr: &str, port: u16) -> Result<(), TransportError>**:
   - TcpListener::bind(format!("{bind_addr}:{port}"))
   - Log at info!: "listening on {addr}"
   - Loop: accept connection, log at warn!: "accepted connection from {peer}"
   - apply_keepalive on accepted stream
   - Handle connection in a loop (NOT tokio::spawn — single client at a time is fine for v1):
     - recv_frame in a loop
     - Log received frame payload size at info! level
     - On ConnectionClosed: log at warn! "client disconnected", break inner loop, go back to accept
     - On other errors: log at warn!, break inner loop

   **Tailscale IP auto-detection** (private helper):
   - `async fn resolve_tailscale_ip() -> Result<String, TransportError>`: Run `tailscale ip -4` via tokio::process::Command, parse stdout, trim whitespace
   - Used by server when --bind is not provided

   **pub async fn client(remote_addr: &str, port: u16, mut frame_rx: tokio::sync::mpsc::Receiver<Frame>) -> Result<(), TransportError>**:
   - Reconnect loop (loop forever, never give up):
     - TcpStream::connect(format!("{remote_addr}:{port}"))
     - On success: log at warn! "connected to {addr}", apply_keepalive, reset backoff to 1.0
     - Enter send loop: recv from frame_rx channel, call send_frame
     - On any send error: log at warn! "connection lost: {err}", break send loop
     - On connect error: log at warn! "connect failed: {err}; retrying in {backoff:.1}s"
   - Backoff after each failure:
     - jitter = rand::thread_rng().gen_range(0.0..backoff * 0.25)
     - sleep(backoff + jitter)
     - backoff = (backoff * 2.0).min(30.0)
   - Start backoff at 1.0, cap at 30.0
   - Reset backoff = 1.0 immediately on successful connection (before entering send loop)

   **Important:** Remove `#![allow(dead_code)]` from protocol.rs since transport.rs now uses Frame and FrameError.

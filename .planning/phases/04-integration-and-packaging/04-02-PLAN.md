  <what-built>
    Complete clipboard bridge: `cssh local` watches clipboard for screenshots and sends them over TCP to `cssh remote`, which writes them to the X11 clipboard via xclip. The `cssh setup` subcommand installs systemd user services and prints a shell snippet for DISPLAY sourcing.
  </what-built>
  <how-to-verify>
**Setup (one-time):**

1. On BOTH machines, install the binary:
   ```bash
   cd /path/to/codex-screenshot-ssh
   cargo install --path .
   ```

2. On the LOCAL machine:
   ```bash
   cssh setup local --remote <tailscale-ip-of-remote>
   ```
   Confirm: "Service started" message appears.

3. On the REMOTE machine:
   ```bash
   cssh setup remote --bind <tailscale-ip-of-remote>
   ```
   Confirm: "Service started" message appears.

4. On the REMOTE machine, add the printed shell snippet to `~/.bashrc` and `~/.zshrc`, then open a NEW SSH session.

5. In the new SSH session, verify:
   ```bash
   echo $DISPLAY
   ```
   Should print `:1` (or similar `:N`).

**E2E Test — Claude Code (E2E-01):**

6. On the LOCAL machine, take a screenshot (Flameshot, gnome-screenshot, PrtScn, etc.)
7. On the REMOTE SSH session, open Claude Code
8. Press Ctrl-V
9. **Expected:** `[Image #1]` appears within a few seconds

**E2E Test — Codex (E2E-02):**

10. Take a new screenshot on LOCAL
11. On REMOTE, open Codex
12. Press Ctrl-V
13. **Expected:** Screenshot image appears

**E2E Test — OpenCode (E2E-03):**

14. Take a new screenshot on LOCAL
15. On REMOTE, open OpenCode
16. Press Ctrl-V
17. **Expected:** Screenshot image appears

**Bonus — Multiple pastes:**

18. After the first paste, take another screenshot on LOCAL
19. Press Ctrl-V again in any CLI tool
20. **Expected:** Second screenshot appears (not the first one)

**If anything fails:**

- Check service logs: `journalctl --user -u cssh-local -f` / `journalctl --user -u cssh-remote -f`
- Verify DISPLAY: `echo $DISPLAY` in SSH session
- Verify xclip: `xclip -selection clipboard -t image/png -o | file -` should show PNG data
- Check connection: service logs should show "new connection" / "connected" messages

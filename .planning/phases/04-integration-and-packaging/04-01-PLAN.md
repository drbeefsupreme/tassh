Add a `Setup` variant to the existing `Commands` enum with a nested `#[command(subcommand)]` field of type `SetupTarget`. Define:

```rust
Setup {
    #[command(subcommand)]
    target: SetupTarget,
},
```

Define `SetupTarget` enum with `Local(SetupLocalArgs)` and `Remote(SetupRemoteArgs)`:
- `SetupLocalArgs`: `--remote` (String, required), `--port` (u16, default 9877)
- `SetupRemoteArgs`: `--bind` (String, required), `--port` (u16, default 9877)

**In src/setup.rs (new file):**

1. `fn home_dir() -> PathBuf` — reads `std::env::var("HOME")`, returns PathBuf.

2. `fn binary_path() -> PathBuf` — returns `home_dir().join(".cargo/bin/cssh")`.

3. `fn unit_dir() -> PathBuf` — returns `home_dir().join(".config/systemd/user")`.

4. `fn cssh_local_unit(remote: &str, port: u16) -> String` — generates unit file content:
   ```ini
   [Unit]
   Description=cssh local clipboard relay
   After=network.target

   [Service]
   ExecStart=/home/user/.cargo/bin/cssh local --remote {remote} --port {port}
   Restart=always
   RestartSec=5

   [Install]
   WantedBy=default.target
   ```
   ExecStart uses `binary_path().display()` for the absolute path. Parse `remote` — if it contains `:`, split host:port and use those; otherwise use `remote` and `port` as separate args.

5. `fn cssh_remote_unit(bind: &str, port: u16) -> String` — same pattern but for `cssh remote --bind {bind} --port {port}`.

6. `fn shell_snippet() -> &'static str` — returns:
   ```
   # cssh: auto-export DISPLAY in SSH sessions
   if [ -n "$SSH_CONNECTION" ] && [ -f "$HOME/.cssh/display" ]; then
       . "$HOME/.cssh/display"
   fi
   ```

7. `fn run_systemctl(args: &[&str]) -> anyhow::Result<()>` — runs `systemctl` with the given args, checks exit status, returns error on failure.

8. `fn run_setup(service_name: &str, unit_content: &str) -> anyhow::Result<()>`:
   - Create `unit_dir()` with `create_dir_all`
   - Write unit content to `unit_dir().join(service_name)`
   - Print "Wrote {path}"
   - Run `systemctl --user daemon-reload`
   - Run `systemctl --user enable {service_name}`
   - Run `systemctl --user start {service_name}`
   - Run `loginctl enable-linger` (no args) — on failure, print warning to stderr but do NOT abort
   - Print the shell snippet with instructions
   - Print status hint: `journalctl --user -u {service_name} -f`

9. `pub fn run_setup_local(args: &SetupLocalArgs) -> anyhow::Result<()>`:
   - Verify binary exists at `binary_path()` — error with helpful message if missing ("Run `cargo install --path .` first")
   - Call `run_setup("cssh-local.service", &cssh_local_unit(&args.remote, args.port))`

10. `pub fn run_setup_remote(args: &SetupRemoteArgs) -> anyhow::Result<()>`:
    - Same pattern for remote
    - Call `run_setup("cssh-remote.service", &cssh_remote_unit(&args.bind, args.port))`

**In src/main.rs:**
Add `mod setup;` declaration. Add `Commands::Setup { target }` match arm that calls `setup::run_setup_local` or `setup::run_setup_remote` based on `SetupTarget` variant. On error, print and exit(1).

**In src/lib.rs:**
Add `pub mod setup;` for test access.

Use `anyhow` for error handling (already a dependency). Do NOT add any new crate dependencies.

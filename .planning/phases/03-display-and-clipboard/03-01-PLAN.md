<task type="auto">
  <name>Task 2: Implement DisplayManager in src/display.rs</name>
  <files>src/display.rs</files>
  <action>
Replace the stub in `src/display.rs` with a complete DisplayManager implementation. The module must provide:

**Public API:**
- `pub struct DisplayManager` with fields: `pub env: DisplayEnvironment`, `pub display_str: String`, and private `xvfb_child: Option<tokio::process::Child>`
- `pub async fn detect_and_init() -> anyhow::Result<DisplayManager>` — the main entry point
- `pub async fn shutdown(mut self)` — clean shutdown (kill Xvfb, remove ~/.cssh/display)

**Detection logic in detect_and_init():**
1. Check `$WAYLAND_DISPLAY` env var — if set and non-empty, return `DisplayEnvironment::Wayland`
2. Check `$DISPLAY` env var — if set and non-empty, return `DisplayEnvironment::X11`
3. Neither found → headless path: clean stale locks, spawn Xvfb, publish display file

**Xvfb spawn (headless path):**
- Create a pipe using `libc::pipe()` (NOT pipe2 with O_CLOEXEC — the write fd must be inherited by child)
- Spawn `Xvfb` with args `["-displayfd", &write_fd_str, "-screen", "0", "1x1x24"]` using `tokio::process::Command`
- Close the write fd in the parent after spawn (so read doesn't hang)
- Read the display number from the read fd (Xvfb writes "N\n" to the fd)
- Set `std::env::set_var("DISPLAY", &format!(":{display_num}"))` in the current process
- Call `publish_display()` to write `~/.cssh/display`
- Return with `DisplayEnvironment::Xvfb`

**Stale lock file cleanup (`clean_stale_xvfb_locks`):**
- Iterate display numbers 0..100
- For each `/tmp/.X{N}-lock` that exists, read the PID inside (decimal string, trimmed)
- Check if PID is alive using `libc::kill(pid as libc::pid_t, 0) == 0`
- If process is dead: remove `/tmp/.X{N}-lock` and `/tmp/.X11-unix/X{N}`
- Log each removal at info level: "removed stale Xvfb lock: /tmp/.X{N}-lock (dead PID {pid})"

**Display file management:**
- `publish_display(display_str)`: Create `~/.cssh/` dir if absent, write `export DISPLAY={display_str}\n` to `~/.cssh/display`
- `display_file_path()`: Returns `$HOME/.cssh/display` (fall back to `/root` if HOME unset)

**shutdown():**
- If `xvfb_child` is Some, call `child.kill().await`
- Remove `~/.cssh/display` via `std::fs::remove_file`
- Log at info level: "Xvfb stopped, {path} removed"

**Xvfb crash auto-restart with backoff:**
- After spawning Xvfb, start a background `tokio::spawn` task that calls `child.wait().await`
- On unexpected exit: log warning, increment attempt counter, sleep with exponential backoff (2s, 4s, 8s, 16s, 32s)
- Re-spawn Xvfb with same `-displayfd` pattern
- After 5 failed attempts, log error and call `std::process::exit(1)`
- Use an `Arc<Mutex<Option<Child>>>` or channel to communicate the new child handle back

**Logging:** Use `tracing::info!` for normal events (display detected, Xvfb spawned, lock cleaned). Use `tracing::debug!` for verbose details (detection steps, individual lock file checks). Use `tracing::warn!` for Xvfb crashes. Use `tracing::error!` for fatal failures.

**Import `DisplayEnvironment` from `crate::protocol`** — it already has the Wayland, X11, Xvfb, Headless variants.

**Error handling:** Use `anyhow::Result` for fallible operations. If Xvfb binary is not found, fail with a clear message: "Xvfb not found. Install with: apt install xvfb"

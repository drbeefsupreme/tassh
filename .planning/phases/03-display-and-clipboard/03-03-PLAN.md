Update the `Commands::Local` arm in main.rs to spawn the clipboard watcher and feed frames to the transport client.

**Current state:** The local arm creates an mpsc channel but drops `tx` unused. The channel feeds into `transport::client()`.

**New wiring:**
1. Remove the `let _tx = tx;` line
2. Spawn `clipboard::watch_clipboard(tx)` as a background tokio task:
   ```rust
   let watch_handle = tokio::spawn(async move {
       if let Err(e) = clipboard::watch_clipboard(tx).await {
           tracing::error!("clipboard watcher error: {e}");
       }
   });
   ```
3. The existing `transport::client(&remote_host, port, rx).await` stays — it consumes frames from the channel

4. Use `tokio::select!` to run both the client transport and handle Ctrl-C:
   ```rust
   tokio::select! {
       result = transport::client(&remote_host, port, rx) => {
           if let Err(e) = result { eprintln!("transport error: {e}"); std::process::exit(1); }
       }
       _ = tokio::signal::ctrl_c() => {
           tracing::info!("Ctrl-C received, shutting down");
       }
   }
   ```

   - Option B: Add a converter between the two channels
   - **Choose Option A** — simpler, clipboard module already imports from protocol. Update `watch_clipboard` signature to `Sender<protocol::Frame>` and wrap PNG bytes in `Frame::new_png()` before sending.

**Add imports:** `use crate::clipboard;`, `use crate::display;`

**main.rs — Remote arm:**

2. Call `clipboard::check_clipboard_tools(&display_mgr.env).await?` to fail fast if xclip/wl-copy missing

4. Use `tokio::select!` to run server and handle signals:
   ```rust

       result = transport::server(&bind_addr, args.port, &display_mgr) => {

       }
       _ = sigterm.recv() => {
           tracing::info!("SIGTERM received, shutting down");
       }
       _ = tokio::signal::ctrl_c() => {
           tracing::info!("Ctrl-C received, shutting down");
       }
   }

   ```
